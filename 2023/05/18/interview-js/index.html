1. JS

数据类型

js 的数据类型有哪些？

  • 基本数据类型
    • number、string、boolean、null、undefined、Symbol、BigInt
  • 引用数据类型（统称 object 类型）
    • 细分：Object、Array、Function、Date

为什么 0.1+0.2 > 0.3?

  • 因为 js 的底层中，每个变量是以二进制表示，固定长度是 64 位，其中第 1 位是符号位，再往后 15 位是指数位，最后 52 位是尾数位，而 0.1 和 0.2 转成二进制时是无限循环小数，js 进行了截取，所以此时的数值并不是原来的值，会比原来的值大

  • 怎么解决？同时放大倍数，再除以相应的倍数

数据类型的判断？

  • typeof
    • typeof null 的值是 object，无法区分 null 还是 object
  • instanceof
    • 只能判断某对象是否存在域目标对象的原型链上
  • constructor
  • Object.prototype.toString.call()
    • 它可以区分 null 、 string 、boolean 、 number 、 undefined 、 array 、 function 、 object 、 date 、 math 数据类型
    • 无法区分为谁的实例

instanceof 的原理

  • 实际上就是查找目标对象的原型链

为什么 typeof null 是 Object?

  • 因为在 js 底层中，所有对象都是二进制存储的，如果二进制前三位都是 0 的话，系统会判定为 object 类型，而 null 二进制全是 0

== 和 ===有什么区别？

  • 前者是非严格意义上的全等
  • 后者是严格意义上的全等，需要同时满足值的大小和数据类型相等

NaN=== NaN 返回什么？

  • 返回 false，NaN 永远不等于 NaN，判断是否为 NaN 用一个函数 isNaN 来判断，isNaN 传入的如果是其他数据类型，那么现将它使用 Number()转为数字类型在进行判断

call、apply、bind 的区别：

  • 作用都是相同的，都是动态的修改当前函数内部环境对象 this 的指向

  • call 与 apply 的主要区别在于参数列表的不同，其它作用是一样的

  • bind 主要是返回一个函数的副本并指定参数，并不会立即运行函数。

执行栈和执行上下文

  • 什么是作用域？什么是作用域链？

    • 规定变量和函数的可使用范围称为作用域
    • 每个函数都有一个作用域链，查找变量或函数时，需要从局部作用域到全局作用域依次查找，这些作用域的集合称作作用域链

  • 什么是执行栈？什么是执行上下文？

    • 执行栈

      • 首先栈特点：先进后出
      • 当进入一个执行环境，就会创建出它的执行上下文，然后进行压栈，当程序执行完成时，它的执行上下文就会被销毁，进行弹栈
      • 栈底永远是全局环境的执行上下文，栈顶永远是正在执行函数的执行上下文
      • 只有浏览器关闭的时候全局执行上下文才会弹出

    • 执行上下文

      • 全局执行上下文
      • 函数执行上下文
      • eval 执行上下文

什么是闭包？闭包的优缺点？闭包的应用？

  1. 闭包是嵌套的内部函数，包含被引用变量（函数）的对象

  • 优点：

    • 可以读取函数内部的变量

    • 将局部变量保存在内存中，实现数据的共享

    • 可以避免使用全局变量，避免全局变量污染

  • 缺点：

    • 被引用的私有变量不能被销毁，增大了内存消耗，滥用闭包可能导致内存泄露，解决方法是可以在使用完变量后手动为它赋值为 null
    • 闭包涉及跨域访问，所以会导致性能损失，我们可以通过把跨作用域变量存储在局部变量中，然后直接访问局部变量，来减轻对执行速度的影响
  1. 应用
    • 模拟私有方法
    • setTimeout 的循环
    • 匿名自执行函数
    • 结果要缓存场景

内存泄露、垃圾回收机制：

  • 什么是内存泄漏？
    • 指不再使用的内存没有被释放，无法使用这段内存
  • 为什么会导致内存泄漏？
    • 指 js 无法访问这个对象，垃圾回收机制却认为该对象还在使用，因此垃圾回收机制不会释放对象，导致该内存不会被释放，越积越多，最终导致系统崩溃。
  • 垃圾回收机制有什么方法？
    • 标记清除法：垃圾回收机制获取根并标记他们，然后访问并标记所有来自它们的引用，然后在访问这些对象并标记它们的引用…如此递进结束后若发现有没有标记的（不可达的）进行删除，进入执行环境的不能进行删除
    • 引用计数法

单线程

  • 为什么 js 是单线程的？
    • 因为 js 里面有可视的 dom，如果是多线程的话，这个线程正在删除 dom 节点，另一个线程正在编辑 dom 节点，导致浏览器不知道处理哪个线程。

同步异步

  • js 任务分为同步和异步任务，异步任务又分成宏任务和微任务。
    • 宏任务：script、setTimeout、setInterval、setImmediate、i/o 操作（输入输出，比如读取、文件操作、网络请求）、ui render（dom 渲染，即更改代码重新渲染 dom 的过程）、异步 ajax 等
    • 微任务：Promise（then、catch、finally）、async/await、process.nextTick、Object.observe(⽤来实时监测 js 中对象的变化)、 MutationObserver(监听 DOM 树的变化)
  • 执行顺序
    • js 代码在执行的时候，会先执行同步代码
    • 执行宏任务 script，
    • 进入 script 后，所有的同步任务主线程执行
    • 所有宏任务放入宏任务执行队列
    • 所有微任务放入微任务执行队列
    • 先清空微任务队列，
    • 再取一个宏任务，执行，再清空微任务队列
    • 依次循环

变量和函数怎么进行提升的？优先级是怎么样的？

  • 对变量提升，只声明，不赋值，值为 undefined
  • 对函数的提升
    • 函数声明的提升，引用类型的赋值
    • 函数表达式的提升，此种声明方式我们可以理解为一个普通变量的提升，将会报错

var let const 有什么区别？

  • var
    • var 可以提升变量声明，let const 不能
    • var 可以重复声明，let const 不能
    • var 在非函数作用域中是挂载到 window 上的
  • let
    • 声明的变量只在局部作用域起作用
    • 防止变量污染
  • const
    • 具有 let 的特征
    • 必须赋初值
    • 不允许改变指针的指向

如何理解模块化？为什么要使用模块化？都有哪几种方式可以实现模块化，各有什么特点？

  • 如何理解模块化？

    • 对于一个复杂的程序,将其按照一定的规范封装成几个文件块,每一块向外暴露一些接口, 但是块的内部数据是私有的, 块与块之间通过暴露的接口进行通信，这个过程称为模块化。

  • 为什么要使用模块化？

    • 防止命名冲突
    • 更好的分离，按需加载
    • 提高的复用性
    • 提高的维护性

  • commonJs amd umd cmd esm

    • commonjs 是同步加载模块，适合服务端
    • amd umd cmd 是异步执行，适合前端
    • esm 使用 export 、 export default 来导出模块，使用 import 来引入模块

  • ESM 和 commonjs 的区别主要在于：

    • commonjs 是运行时加载，esm 是编译时加载
    • commonjs 是同步加载模块，esm 是异步加载模块

  • require 和 import 的区别

    • 调用时机
      • require 是运行时调用，所以可以放在任何地方
      • import 是编译时调用，所以只能放在文件开头
    • 使用时
      • require 用 module.exports = vaule 或 exports.xxx = vaule
      • import 用（ export defualt ）默认暴露，（export const xxx()）统一暴露，(export {xx,xxx})分别暴露
    • require 是赋值的过程
    • import 是解构的过程

箭头函数和普通函数的区别？箭头函数可以当做构造函数 new 吗？

  • 箭头函数是普通函数的简写，但是它不具备很多普通函数的特性
  • 第一点，this 指向问题，箭头函数的 this 指向它定义时所在的对象，而不是调用它的对象
  • 不会进行函数提升
  • 没有 arguments 对象，不能使用 arguments，如果要获取参数的话可以使用 rest 运算符
  • 没有 yield 属性，不能作为生成器 Generator 使用
  • 不能 new
    • 没有自己的 this，不能调用 call 和 apply
    • 没有 prototype，new 关键字内部需要把新对象的proto指向函数的 prototype

什么是防抖？什么是节流？

  • 防抖：n 秒后执行该事件，若 n 秒内重复触发，则重新计时

  • 节流：n 秒内执行一次事件，若 n 秒内重复触发，只执行一次

js 遍历数组的方法

  • for…in 、for…of、map、forEach、filter、every、some、reduce

    • for…in：遍历对象

    • for…of：遍历数组或对象

    • map：映射，返回一个新数组，不改变原数组

    • forEach：array.forEach()

    • filter：

      用于对数组进行过滤。
      filter() 方法创建一个新的数组，新数组中的元素是通过检查指定数组中符合条件的所有元素。

      filter() 不会对空数组进行检测；不会改变原始数组

    • every：返回一个布尔值，所有元素满足条件，才返回 true，有一个元素不满足则返回 false，停止遍历

    • some：返回一个布尔值，有一个元素满足条件，就返回 true

    • reduce：reduce(pre, cur, index, array), init)

      • pre（必须） 初始值
      • cur （必须）当前元素
      • index 当前元素的索引
      • array 当前所选元素的对象
      • init 传递给元素的初始值，相当于 pre 的初始值

改变原数组的方法

  • pop、push、 shift 、unshift 、sort 、reverse 、splice
    • pop：用于删除并返回最后一个值
    • push：用于数组的尾部新加一个或多个值，返回新数组的长度
    • shift：用于删除并返回第一个值
    • unshift：用于数组的头部新加一个或多个值，返回新数组的长度
    • sort：排序，从小到大或从大到小 (a, b) => a - b 或 (a, b) => b - a
    • reverse：倒序，将数组的值颠倒排序
    • splice： 截取数组的一部分，传入两个参数，第一个参数是开始的索引，第二个参数是截取的长度，返回截取的内容

不改变数组的方法

  • concat、join、slice、filter、map、find
    • concat：合并数组，用于连接两个或多个数组
    • join：用于把数组中的所有元素放入一个字符串，元素之间通过制定的分隔符进行分割
    • slice：截取数组，传入两个参数，第一个参数是开始的索引，第二个参数是结束的索引
    • filter：过滤数组，返回为真的，放入一个新的数组
    • map：在原来的数组上进行修改，返回一个新的数组
    • find：返回为真的第一项，后面有真也不继续找

web worker 的用处？

  • js 是单线程语言，web worker 可以为 js 创建多个子线程，这些子线程不会阻塞主线程。但是有几个限制：
    • 同源限制：子线程和主线程必须是同源
    • dom 限制：子线程所在的全局对象和主线程的不一样，所以不能对 dom 操作，但可以对 navigator 和 location 对象
    • 文件限制：不能打开本机的（file:/）,必须来自网络
    • 通信限制：每个线程都有独立的运行机制，线程之间的通信需要借助 postMessage
    • 脚本限制：不能使用 alert、confirm，但可以使用 XMLHttpRequest 发出 Ajax 请求

clientHeight、scrollHeight、offsetHeight 的区别

  • scrollHeight = 内容高度 + padding
  • clientHeight = 可见高度 + padding
  • offsetHeight = 可见高度 + padding + border + 滚动条高度

Dom、Bom 的区别

  • DOM 是指文档对象模型，它是一种表示文档的标准对象集合，可以通过 JavaScript 来访问和操作。DOM 定义了 HTML、XML 和 SVG 文档的结构，以及访问和操作这些文档的方法和属性
  • BOM 是指浏览器对象模型，它是浏览器提供的一组对象集合，用于访问和控制浏览器窗口和页面之外的部分，例如 history、location、screen、navigator

倒计时用 setimeout 还是 setInterval？

  • 用 setTimeout，因为当程序执行时间大于定时器的时间时，在程序没执行完的时间内会直接执行下一次函数
  • settimeout 和 setinterval 都是 JavaScript 中的定时器函数
  • settimeout 是在指定时间后执行一次函数，而 setinterval 是每隔一定时间执行一次函数
  • 一般来说，settimeout 适用于只需要执行一次的任务，而 setinterval 适用于需要重复执行的任务。但是需要注意的是，使用 setinterval 时要确保函数执行的时间不会超过间隔时间，否则会出现累积效应，导致执行次数增多

promise、async/await

  • async/await
    • 优点：
      • 做到串行同步写法，代码阅读相对容易
      • 对判断语句友好，可以直接写在判断语句里面
      • 对于复杂的流程，逻辑相对清晰
    • 缺点：
      • 用 await 可能导致性能问题，因为 await 会阻塞代码
  • promise 相对于 async…await 的缺点：
    • 无法取消
    • 不能用 try…catch 捕获异常

fetch 的优缺点？

  • fetch 脱离了 xhr，基于 promise 实现
  • 对于某些错误不会 reject，如状态码 400，500
  • 不支持超时处理
  • 默认不携带 cookie，需要手动配置
  • 不能监测请求进度，而 xhr 可以

  • input type = ‘file’接收

秒传、切片上传、断点续传

  • 秒传：是文件上传之前计算其内容的 hash 值，将该值传到后台，如果后台存在 hash 值一致的文件，就代表传输完成
  • 切片上传：利用 Blob 的 slice 方法，将大文件切割成若干个小文件分别传输，上传完成时由服务端整合
  • 断点续传：在分片上传的基础上，分成一个个小文件之后，每个小文件上传完毕之后对其进行状态的存储（localStorage），如果中间发生网络断线或者刷新，下次可以接着上次的进度上传

e.target 和 e.currentTarget 的区别：

  • e.target 是点击的对象，e.currentTarget 是绑定事件的对象

js 性能优化：

  • 慎用全局变量
    • 全局作用域链长，生命周期长，查找时间有所消耗
    • 如果某个局部作用域出现同名变量，会导致全局污染或遮蔽
  • 缓存全局变量
    • 将使用中无法避免的全局变量缓存到局部
  • 尽少的使用闭包
    • 闭包提供一些便捷性，但也会有一些性能的影响，由于保留着原应被收回的变量引用，增长了作用域链，影响性能
    • 会有内存泄漏的风险
  • 使用字面量新建对象
  • 使用性能更优的循环遍历(耗时从少到多)
    • for -> forEach / for…of -> for…in
  • 避免使用 with 和 eval
    • with 可以改变当前的作用域环境，将对象推向作用域链的头部，这使得作用域环境内的局部变量的访问效率降低
    • eval 将传入的字符串当作脚本执行，大幅度降低脚本执行性能
