一. HTML/CSS

Flex：弹性布局（一维）

  1. 基本概念：容器/项目，主轴/交叉轴，起始线/结束线

  2. 容器属性：

  • flex-direction（主轴方向）

    • row（默认值）：主轴为水平方向，起点在左端。
    • row-reverse：主轴为水平方向，起点在右端。
    • column：主轴为垂直方向，起点在上沿。
    • column-reverse：主轴为垂直方向，起点在下沿。

  • flex-wrap（换行）

    • nowrap：默认值，不换行
    • wrap：换行，第一行在上方
    • wrap-reverse：换行，第一行在下方

  • flex-flow（direction 和 wrap 的组合）

  • justify-content（定义项目在主轴上对齐的方式）

    • flex-start：左对齐
    • flex-end：右对齐
    • center：居中
    • space-between：两端对齐，项目之间间隔相等
    • space-around：每个项目两侧间隔相等

  • align-items（定义项目在交叉轴上对齐的方式）

    • stretch：默认值，如果项目未设置高度或者设为 auto，自动占满整个容器的高度
    • flex-start：交叉轴的起点对齐
    • flex-end：交叉轴的终点对齐
    • center：交叉轴的中心对齐
    • baseline：项目的第一行文字基线对齐

  • align-content（定义了多根轴线的对齐方式）

    • flex-start：与交叉轴的起点对齐。

    • flex-end：与交叉轴的终点对齐。

    • center：交叉轴的中点对齐。

    • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。

    • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。

    • stretch（默认值）：轴线占满整个交叉轴。

  1. 项目属性：

    • order：定义项目的排序，数值越小，排列越靠前，默认为 0

    • flex-grow：剩余空间充足时，定义项目放大比例，，默认为 0

    • flex-shrick：剩余空间不足时，定义项目缩小比例，默认为 1

    • flex-basis：定义了在分配多余空间之前，项目占据的主轴空间（main size）

    • flex：flex-grow、flex-shrick、flex-basis 的简写

    • align-self：允许单个项目有与其他项目不一样的对齐方式，可覆盖 align-item 属性

Gird：网格布局（二维）

行内元素、块级元素、行内块元素

  1. 行内元素：inline

    • 不能设置宽高，设置 margin、padding 不起效果，可以设置 line-height 宽高由内容决定

    • 常见元素：span、input、img、textarea、label、select

  2. 块级元素：block

    • 能设置宽高，独占一行

    • 常见元素：p、h1/h2/h3/h4/h5、div、ul、li、table

  3. 行内块元素：inline-block

    • 能设置宽高，多个元素共享一行

请说明 px,rpx,em,rem,vw,vh 等单位的特性

  • px：像素，绝对长度单位
  • rpx：微信小程序独有，解决屏幕自适应的尺寸单位
  • em：相对单位，以父元素为基准
  • rem：相对单位，以根元素为基准
  • vw / vh：相对于视窗的宽高

常见替换元素、非替换元素

  • 替换元素
    • 浏览器根据标签的属性来判断具体显示内容
    • img、input、textarea
  • 非替换元素
    • p、div、span

first-child 和 first-of-child 有什么区别？

  • first-child：所有标签中的第一个,并且要符合设定的标签类型
  • first-of-child：所有符合设定类型的标签中的第一个

doctype 标签和 meta 标签

  • doctype：告诉浏览器以哪种声明文档规范解析文档
  • meta：放在 head 标签内，用来提供关于 HTML 文档的一些元信息

script 标签中的 defer 和 async

  1. 问题来源：script 加载会阻塞页面的加载，如果我们引用外部 js，在网络不佳的情况下，就会导致页面长期处于白屏状态。

  2. 解决方法：script 标签 defer、async

    • defer：

      • 不会阻止页面的解析，并下载此脚本，当页面解析完成后，立即执行下载的脚本（若需要下载多个脚本，则等待其他脚本加载完成，再执行脚本），脚本执行完成后才会触发 DOMContentLoaded 事件

    • async：

      • 不阻塞浏览器解析 HTML，但是 script 下载完成后，会立即中断浏览器解析 HTML，并执行此 script
      • 互相独立，谁先下载完，谁先执行，没有固定的先后顺序，不可控

  3. 最大区别在于执行时机

什么是 BFC？BFC 有什么特性？如何创建 BFC？BFC 的应用场景？

  1. BFC 是块级格式化上下文，是在盒模型下对块级元素进行布局的独立渲染区域，该区域内的元素和区域外的元素互不影响
  2. 特性：
    • 同一个 BFC 下 margin 会重叠
    • BFC 内部的元素不会影响外部的元素
    • BFC 是一个块级元素，块级元素在垂直方向上依次排列
    • BFC 区域不会与 float 元素重叠
  3. 如何创建
    • 定位元素：position设置为absolute或者flexed
    • 浮动元素：float不为none
    • overflow：overflow设置为hidden
    • 布局：inline-block、flexed
  4. 应用场景：
    • 防止两个相邻块级元素的上下 margin 值发生重叠（例：上下 margin 重合）
    • 清除浮动（例：父元素没有设置高度，子元素浮动溢出）
    • 实现自适应布局，防止元素被浮动元素覆盖（左边固定，右边自适应）

如何清除浮动

  1. 额外标签clear:both

  2. 利用 BFC

    缺点：如果有内容要溢出显示，也会被一同隐藏

什么是 DOM 事件流？什么是事件委托？

  1. 事件发生时会在元素节点间按照特定的顺序传播,这个传播过程即 DOM 事件流。
    • dom 事件流分为三个阶段：捕获阶段、目标阶段、冒泡阶段
    • addEventListener (type, listener,[useCapture] )，第三个参数不写默认是 false，false 表示在事件冒泡阶段，如果是true，表示事件在捕获阶段
  2. 事件委托
    • 不给每个子节点单独设置事件监听器，而是给父节点设置事件监听器，利用事件冒泡原理影响设置每个子节点
    • 优点：减少内存消耗，提高性能

什么是事件捕获？什么是事件冒泡？

  1. 事件捕获
    • 自上而下执行
    • 事件捕获用意在于事件到达预定目标之前捕获它
  2. 事件冒泡
    • 自下而上执行
    • 开始由某个具体的事件接受，然后逐级向上传播到最顶层的节点
    • 阻止事件冒泡：e.stopPropagation或者cancelBubble = true

link 标签和 import 标签的区别

  • link 属于 html，@import 属于 css
  • link 和页面同时加载，@import 引用的 css 在页面加载结束后加载
  • link 属于 html 标签，没有兼容性，@import 只有 IE5 以上兼容
  • link 权重高于@import

css 优先级问题

标准盒模型和怪异盒模型

二. JS

数据类型

js 的数据类型有哪些？

  • 基本数据类型
    • number、string、boolean、null、undefined、Symbol、BigInt
  • 引用数据类型（统称 object 类型）
    • 细分：Object、Array、Function、Date

为什么 0.1+0.2 > 0.3?

  • 因为 js 的底层中，每个变量是以二进制表示，固定长度是 64 位，其中第 1 位是符号位，再往后 15 位是指数位，最后 52 位是尾数位，而 0.1 和 0.2 转成二进制时是无限循环小数，js 进行了截取，所以此时的数值并不是原来的值，会比原来的值大

  • 怎么解决？同时放大倍数，再除以相应的倍数

数据类型的判断？

  • typeof
    • typeof null 的值是 object，无法区分 null 还是 object
  • instanceof
    • 只能判断某对象是否存在域目标对象的原型链上
  • constructor
  • Object.prototype.toString.call()
    • 它可以区分 null 、 string 、boolean 、 number 、 undefined 、 array 、 function 、 object 、 date 、 math 数据类型
    • 无法区分为谁的实例

instanceof 的原理

  • 实际上就是查找目标对象的原型链

为什么 typeof null 是 Object?

  • 因为在 js 底层中，所有对象都是二进制存储的，如果二进制前三位都是 0 的话，系统会判定为 object 类型，而 null 二进制全是 0

== 和 ===有什么区别？

  • 前者是非严格意义上的全等
  • 后者是严格意义上的全等，需要同时满足值的大小和数据类型相等

NaN=== NaN 返回什么？

  • 返回 false，NaN 永远不等于 NaN，判断是否为 NaN 用一个函数 isNaN 来判断，isNaN 传入的如果是其他数据类型，那么现将它使用 Number()转为数字类型在进行判断

call、apply、bind 的区别：

  • 作用都是相同的，都是动态的修改当前函数内部环境对象 this 的指向
  • call 与 apply 的主要区别在于参数列表的不同，其它作用是一样的
  • bind 主要是返回一个函数的副本并指定参数，并不会立即运行函数。

执行栈和执行上下文

  • 什么是作用域？什么是作用域链？

    • 规定变量和函数的可使用范围称为作用域
    • 每个函数都有一个作用域链，查找变量或函数时，需要从局部作用域到全局作用域依次查找，这些作用域的集合称作作用域链

  • 什么是执行栈？什么是执行上下文？

    • 执行栈

      • 首先栈特点：先进后出
      • 当进入一个执行环境，就会创建出它的执行上下文，然后进行压栈，当程序执行完成时，它的执行上下文就会被销毁，进行弹栈
      • 栈底永远是全局环境的执行上下文，栈顶永远是正在执行函数的执行上下文
      • 只有浏览器关闭的时候全局执行上下文才会弹出

    • 执行上下文

      • 全局执行上下文

      • 函数执行上下文

        每次函数调用时，都会新创建一个函数执行上下文
执行上下文分为创建阶段和执行阶段
    创建阶段：函数环境会创建变量对象：arguments对象（并赋值）、函数声明（并赋值）、变量声明（不赋值），函数表达式声明（不赋值）；会确定this指向；会确定作用域
    执行阶段：变量赋值、函数表达式赋值，使变量对象编程活跃对象

      • eval 执行上下文

什么是闭包？闭包的优缺点？闭包的应用？

  1. 闭包是嵌套的内部函数，包含被引用变量（函数）的对象

  • 优点：

    • 可以读取函数内部的变量

    • 将局部变量保存在内存中，实现数据的共享

    • 可以避免使用全局变量，避免全局变量污染

  • 缺点：

    • 被引用的私有变量不能被销毁，增大了内存消耗，滥用闭包可能导致内存泄露，解决方法是可以在使用完变量后手动为它赋值为 null
    • 闭包涉及跨域访问，所以会导致性能损失，我们可以通过把跨作用域变量存储在局部变量中，然后直接访问局部变量，来减轻对执行速度的影响
  1. 应用
    • 模拟私有方法
    • setTimeout 的循环
    • 匿名自执行函数
    • 结果要缓存场景

内存泄露、垃圾回收机制：

  • 什么是内存泄漏？
    • 指不再使用的内存没有被释放，无法使用这段内存
  • 为什么会导致内存泄漏？
    • 指 js 无法访问这个对象，垃圾回收机制却认为该对象还在使用，因此垃圾回收机制不会释放对象，导致该内存不会被释放，越积越多，最终导致系统崩溃。
  • 垃圾回收机制有什么方法？
    • 标记清除法：垃圾回收机制获取根并标记他们，然后访问并标记所有来自它们的引用，然后在访问这些对象并标记它们的引用…如此递进结束后若发现有没有标记的（不可达的）进行删除，进入执行环境的不能进行删除
    • 引用计数法

单线程

  • 为什么 js 是单线程的？
    • 因为 js 里面有可视的 dom，如果是多线程的话，这个线程正在删除 dom 节点，另一个线程正在编辑 dom 节点，导致浏览器不知道处理哪个线程。

同步异步

  • js 任务分为同步和异步任务，异步任务又分成宏任务和微任务。
    • 宏任务：script、setTimeout、setInterval、setImmediate、i/o 操作（输入输出，比如读取、文件操作、网络请求）、ui render（dom 渲染，即更改代码重新渲染 dom 的过程）、异步 ajax 等
    • 微任务：Promise（then、catch、finally）、async/await、process.nextTick、Object.observe(⽤来实时监测 js 中对象的变化)、 MutationObserver(监听 DOM 树的变化)
  • 执行顺序
    • js 代码在执行的时候，会先执行同步代码
    • 执行宏任务 script，
    • 进入 script 后，所有的同步任务主线程执行
    • 所有宏任务放入宏任务执行队列
    • 所有微任务放入微任务执行队列
    • 先清空微任务队列，
    • 再取一个宏任务，执行，再清空微任务队列
    • 依次循环

变量和函数怎么进行提升的？优先级是怎么样的？

  • 对变量提升，只声明，不赋值，值为 undefined
  • 对函数的提升
    • 函数声明的提升，引用类型的赋值
    • 函数表达式的提升，此种声明方式我们可以理解为一个普通变量的提升，将会报错

var let const 有什么区别？

  • var
    • var 可以提升变量声明，let const 不能
    • var 可以重复声明，let const 不能
    • var 在非函数作用域中是挂载到 window 上的
  • let
    • 声明的变量只在局部作用域起作用
    • 防止变量污染
  • const
    • 具有 let 的特征
    • 必须赋初值
    • 不允许改变指针的指向

如何理解模块化？为什么要使用模块化？都有哪几种方式可以实现模块化，各有什么特点？

  • 如何理解模块化？

    • 对于一个复杂的程序,将其按照一定的规范封装成几个文件块,每一块向外暴露一些接口, 但是块的内部数据是私有的, 块与块之间通过暴露的接口进行通信，这个过程称为模块化。

  • 为什么要使用模块化？

    • 防止命名冲突
    • 更好的分离，按需加载
    • 提高的复用性
    • 提高的维护性

  • commonJs amd umd cmd esm

    • commonjs 是同步加载模块，适合服务端
    • amd umd cmd 是异步执行，适合前端
    • esm 使用 export 、 export default 来导出模块，使用 import 来引入模块

  • ESM 和 commonjs 的区别主要在于：

    • commonjs 是运行时加载，esm 是编译时加载
    • commonjs 是同步加载模块，esm 是异步加载模块

  • require 和 import 的区别

    • 调用时机
      • require 是运行时调用，所以可以放在任何地方
      • import 是编译时调用，所以只能放在文件开头
    • 使用时
      • require 用 module.exports = vaule 或 exports.xxx = vaule
      • import 用（ export defualt ）默认暴露，（export const xxx()）统一暴露，(export {xx,xxx})分别暴露
    • require 是赋值的过程
    • import 是解构的过程

箭头函数和普通函数的区别？箭头函数可以当做构造函数 new 吗？

  • 箭头函数是普通函数的简写，但是它不具备很多普通函数的特性
  • 第一点，this 指向问题，箭头函数的 this 指向它定义时所在的对象，而不是调用它的对象
  • 不会进行函数提升
  • 没有 arguments 对象，不能使用 arguments，如果要获取参数的话可以使用 rest 运算符
  • 没有 yield 属性，不能作为生成器 Generator 使用
  • 不能 new
    • 没有自己的 this，不能调用 call 和 apply
    • 没有 prototype，new 关键字内部需要把新对象的proto指向函数的 prototype

什么是防抖？什么是节流？

  • 防抖：n 秒后执行该事件，若 n 秒内重复触发，则重新计时

  • 节流：n 秒内执行一次事件，若 n 秒内重复触发，只执行一次

js 遍历数组的方法

  • for…in 、for…of、map、forEach、filter、every、some、reduce

    • for…in：遍历对象

    • for…of：遍历数组或对象

    • map：映射，返回一个新数组，不改变原数组

    • forEach：array.forEach()

    • filter：

      用于对数组进行过滤。
      filter() 方法创建一个新的数组，新数组中的元素是通过检查指定数组中符合条件的所有元素。

      filter() 不会对空数组进行检测；不会改变原始数组

    • every：返回一个布尔值，所有元素满足条件，才返回 true，有一个元素不满足则返回 false，停止遍历

    • some：返回一个布尔值，有一个元素满足条件，就返回 true

    • reduce：reduce(pre, cur, index, array), init)

      • pre（必须） 初始值
      • cur （必须）当前元素
      • index 当前元素的索引
      • array 当前所选元素的对象
      • init 传递给元素的初始值，相当于 pre 的初始值

改变原数组的方法

  • pop、push、 shift 、unshift 、sort 、reverse 、splice
    • pop：用于删除并返回最后一个值
    • push：用于数组的尾部新加一个或多个值，返回新数组的长度
    • shift：用于删除并返回第一个值
    • unshift：用于数组的头部新加一个或多个值，返回新数组的长度
    • sort：排序，从小到大或从大到小 (a, b) => a - b 或 (a, b) => b - a
    • reverse：倒序，将数组的值颠倒排序
    • splice： 截取数组的一部分，传入两个参数，第一个参数是开始的索引，第二个参数是截取的长度，返回截取的内容

不改变数组的方法

  • concat、join、slice、filter、map、find
    • concat：合并数组，用于连接两个或多个数组
    • join：用于把数组中的所有元素放入一个字符串，元素之间通过制定的分隔符进行分割
    • slice：截取数组，传入两个参数，第一个参数是开始的索引，第二个参数是结束的索引
    • filter：过滤数组，返回为真的，放入一个新的数组
    • map：在原来的数组上进行修改，返回一个新的数组
    • find：返回为真的第一项，后面有真也不继续找

web worker 的用处？

  • js 是单线程语言，web worker 可以为 js 创建多个子线程，这些子线程不会阻塞主线程。但是有几个限制：
    • 同源限制：子线程和主线程必须是同源
    • dom 限制：子线程所在的全局对象和主线程的不一样，所以不能对 dom 操作，但可以对 navigator 和 location 对象
    • 文件限制：不能打开本机的（file:/）,必须来自网络
    • 通信限制：每个线程都有独立的运行机制，线程之间的通信需要借助 postMessage
    • 脚本限制：不能使用 alert、confirm，但可以使用 XMLHttpRequest 发出 Ajax 请求

clientHeight、scrollHeight、offsetHeight 的区别

  • scrollHeight = 内容高度 + padding
  • clientHeight = 可见高度 + padding
  • offsetHeight = 可见高度 + padding + border + 滚动条高度

Dom、Bom 的区别

  • DOM 是指文档对象模型，它是一种表示文档的标准对象集合，可以通过 JavaScript 来访问和操作。DOM 定义了 HTML、XML 和 SVG 文档的结构，以及访问和操作这些文档的方法和属性
  • BOM 是指浏览器对象模型，它是浏览器提供的一组对象集合，用于访问和控制浏览器窗口和页面之外的部分，例如 history、location、screen、navigator

倒计时用 setimeout 还是 setInterval？

  • 用 setTimeout，因为当程序执行时间大于定时器的时间时，在程序没执行完的时间内会直接执行下一次函数
  • settimeout 和 setinterval 都是 JavaScript 中的定时器函数
  • settimeout 是在指定时间后执行一次函数，而 setinterval 是每隔一定时间执行一次函数
  • 一般来说，settimeout 适用于只需要执行一次的任务，而 setinterval 适用于需要重复执行的任务。但是需要注意的是，使用 setinterval 时要确保函数执行的时间不会超过间隔时间，否则会出现累积效应，导致执行次数增多

promise、async/await

  • async/await
    • 优点：
      • 做到串行同步写法，代码阅读相对容易
      • 对判断语句友好，可以直接写在判断语句里面
      • 对于复杂的流程，逻辑相对清晰
    • 缺点：
      • 用 await 可能导致性能问题，因为 await 会阻塞代码
  • promise 相对于 async…await 的缺点：
    • 无法取消
    • 不能用 try…catch 捕获异常

fetch 的优缺点？

  • 对于某些错误不会 reject，如状态码 400，500
  • 不支持超时处理
  • 默认不携带 cookie，需要手动配置
  • 不能监测请求进度，而 xhr 可以

秒传、切片上传、断点续传

  • 秒传：是文件上传之前计算其内容的 hash 值，将该值传到后台，如果后台存在 hash 值一致的文件，就代表传输完成
  • 切片上传：利用 Blob 的 slice 方法，将大文件切割成若干个小文件分别传输，上传完成时由服务端整合
  • 断点续传：在分片上传的基础上，分成一个个小文件之后，每个小文件上传完毕之后对其进行状态的存储（localStorage），如果中间发生网络断线或者刷新，下次可以接着上次的进度上传

js 性能优化：

  • 慎用全局变量
    • 全局作用域链长，生命周期长，查找时间有所消耗
    • 如果某个局部作用域出现同名变量，会导致全局污染或遮蔽
  • 缓存全局变量
    • 将使用中无法避免的全局变量缓存到局部
  • 尽少的使用闭包
    • 闭包提供一些便捷性，但也会有一些性能的影响，由于保留着原应被收回的变量引用，增长了作用域链，影响性能
    • 会有内存泄漏的风险
  • 使用字面量新建对象
  • 使用性能更优的循环遍历(耗时从少到多)
    • for -> forEach / for…of -> for…in
  • 避免使用 with 和 eval
    • with 可以改变当前的作用域环境，将对象推向作用域链的头部，这使得作用域环境内的局部变量的访问效率降低
    • eval 将传入的字符串当作脚本执行，大幅度降低脚本执行性能

三. Vue

有使用过 Vue 吗？说说你对 Vue 的理解

  • vue 是一个构建数据驱动的渐进式框架，它的目标是通过 API 实现响应式数据绑定和视图更新

说说 vue 的优缺点

  1. 优点
    • 数据驱动视图，对真实 dom 抽象出 virtual dom，并配合 diff 算法、响应式、观察者和异步队列等手段以最小的代价更新 dom，渲染页面
    • 组件化，组件用单文件的形式进行代码的编写，使得我们可以在一个文件里编写 html/css/js
    • 强大的 API 提供的一系列 api 能满足业务开发各类需求
    • 采用虚拟 dom，让 vue 的 ssr(服务器端渲染)先天有利
    • 生命周期钩子函数，选项式的代码组织方式
    • 生态好，社区活跃
  2. 缺点
    • 由于底层采用 Object.defineProperty 实现响应式，不支持 ie8 及以下的浏览器
    • csr（客户端渲染）先天不足，首屏性能问题（白屏）
    • 不利于 seo 检索

描述 vue 生命周期

  1. 什么是 vue 的生命周期？
    • vue 从实例的创建到销毁的过程，就是生命周期，整体上分为创建，挂载，更新，销毁
  2. 细分，每一步分两小步
    • beforCreate
      • 创建之前，data 和 method 还没有
      • 在 beforCreate 和 created 挂载 data 和绑定事件
    • created
      • 创建完成，data 和 method 可以用了
      • 在 created 和 beforMount 之间，如果没有设置 el 则生命周期结束，如果有则继续，el 挂载结束后，根据 templete/outerHTML(el)渲染页面
    • beforMount
      • 在 beforMount 之前已创建虚拟 dom
      • 之后在 mounted 之前，将 vm.$el 替换掉页面中的 el
    • mounted
      • 页面渲染完成
      • 一个生命周期直挂载一次
    • beforUpdate
      • data 变化后，对应组件重新渲染之前
    • updated
      • data 变化后，对应组件重新渲染完成
    • beforDestroy
      • 在实例销毁之前，实例仍然可用，手动撤销监听事件，计时器等
    • destroyed
      • 仅存在 Dom 节点，其他所有东西已自动销毁

vue 双向绑定原理

  • vue 数据双向绑定通过数据劫持结合发布订阅模式，利用 Object.defineProperty()来劫持各个属性的 getter、setter，在数据更新变动时发布消息给订阅者，触发响应的监听回调

MVVM 的原理

  • MVVM 作为绑定的入口，整合 Observer、Compile、Watcher 三者，通过 Observer 来监听数据变化，Compile 解析模板指令，Watcher 建立起 Observer 和 Compile 之间的联系，达到数据更新使得视图更新，视图更新使得数据更新的双向绑定

vue 的发布订阅模式

  • data 中每个数据绑定一个发布者 Dep，这个 Dep 收集了依赖该数据的所有订阅者

  • 在数据变化后，触发相应 setter，从而 Dep 发布通知，通知每个订阅者，做出对应的回调函数

Vue 如何监听数组

  • vue 重写了数组的 push、pop、shift、unshift、splice、sort、reverse 七种方法，调用这些方法后做出回调更新

Vue 为什么采用异步更新？

  • 因为 vue 是组件级的更新，在更新非常频繁的情况下，如果不采用异步更新的话每次都需要重新渲染，非常浪费性能
  • 每次数据更新的时候，vue 会把它放到一个队列中，等到最后会调用 nextTick 方法，nextTick 会清空这个队列
  • nextTick(callback)也可以手动调用，同样会把 callback 回调函数放到队列的最后，保证视图更新完成再被调用

Vue 中的 nextTick

  • 解释：在下次 Dom 更新循环结束之后执行延迟回调，在数据修改之后立即执行使用这个方法，获取更新后的 Dom
  • 应用：
    • 想要在生命周期函数 create()中操作 Dom
    • 在数据改变之后执行的操作，而这个操作需要在 Dom 结构改变后才操作
    • 在使用某个第三方插件时 ，希望在 vue 生成的某些 dom 动态发生变化时重新应用该插件

computed、methods、watch 的区别

  1. computed 是计算属性，基于它的依赖进行缓存，只有依赖发生改变时才会重新求值

  2. methods 是方法，表示一个具体的操作，主要用来书写业务逻辑

  3. watch 用来监听数据的变化，从而进行某些业务逻辑的操作，可以看作是 computed 和 methods 的结合体

  4. 应用：

    • 数据量大，需要缓存的时候用 computed ；每次确实需要重新加载，不需要缓存时用 methods

    • 当我们需要进行数值计算时，并且依赖于其他数据时，应该使用 computed；当我们需要在数据变化时执行异步或者开销较大的操作时，应该使用 watch

vue 开发中的性能优化

  1. 不做响应式：对于不改变的数据，使用 Object.freeze()冻结 data 中的属性，从而阻止数据劫持，提升渲染速度
  2. 图片懒加载：第三方插件 vue-lazyload
  3. 路由懒加载
  4. 事件的销毁：组件销毁时，会自动解绑它的全部指令及监听器，但仅限于组件本身的事件，而对于定时器、addEventListener 注册需要在组件销毁的生命周期钩子中手动销毁或解绑，以免内存泄漏
  5. 第三方插件按需加载
  6. 使用 keep-alive 缓存页面
  7. 在使用 for 的同时避免使用 v-if，vue2 中 for 的优先级比 v-if 高
  8. 使用 v-show 复用 dom

vue-router 的模式

  1. hash 模式：监听 hashchange 实现前端路由，利用 url 的 hash 来模拟 hash，以保证 url 改变的时候，页面不会重新加载
  2. history 模式：整个地址重新加载，可以保存历史记录，方便前进后退，但没有后台配置的话，页面刷新时会出现 404

什么是虚拟 dom?

  • 虚拟 dom 是相对于浏览器所渲染出来的真实 dom 的，它用 js 对象描述 dom 结构。

虚拟 Dom 的优缺点

  1. 优点
    • 减少 dom 操作，减少 dom 回流和重绘
    • 保证性能的下限，虽说性能不是最佳，但是它具备局部更新的能力，所以大部分时候还是比正常的 DOM 性能高很多的
  2. 缺点
    • 首次渲染大量 DOM 时，由于多了一层虚拟 DOM 的计算，会比 innerHTML 插入慢

虚拟 Dom 生成的过程

  • 首先当生命周期走到 created 和 beforMount 之间时，会把 template 模板编译成 render 函数，然后在 render 函数运行时调用 h 函数，h 函数内部调用 vnode 函数生成虚拟 dom，虚拟 dom 首次生成
  • 之后，当数据变化时生成新的虚拟 dom，新旧虚拟 dom 用 diff 算法进行比较

diff 算法

  • diff 算法是指对新旧虚拟节点进行对比，并返回一个 patch 对象，用来存储两个节点不同的地方，最后利用 patch 记录的消息局部更新 DOM
  • 步骤：
    • 比较两个虚拟 dom 树，对根节点 root 进行执行 patch(oldVnode，newVnode)函数，比较两个根节点是否是相同节点。如果不同，直接替换（新增新的，删除旧的）
    • 如果相同，对两个节点执行 patchVnode（oldVnode, newVnode），比较属性，文本，以及子节点。此时，要么新增，要么删除，要么直接修改文本内容。只有当都存在子节点时，并且 oldVnode === newVnode 为 false 时。会执行 updateChildren 函数，去进一步比较他们的子节点。
    • 分情况比较
    • 递归比较子节点
